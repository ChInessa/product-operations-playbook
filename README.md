### 🚨 Восстановление проблемного проекта (Project Recovery)

📍 Полный кейс: [Project Recovery Case](./docs/project-recovery-case.md)

**Ситуация:**
- проект без приёмки со стороны заказчика (0%)
- хаотичное управление задачами (Redmine как список задач)
- ежедневные встречи ~60 минут без результата
- задачи размером до 100–130 часов
- размытая ответственность и узкие места (DevOps, методология)

**Действия:**
- перевод проекта в Jira и внедрение Epics → Stories → Tasks
- разделение команды на 4 кросс-функциональные группы
- внедрение delivery pipeline (от аналитики до production)
- замена общих daily на 15-минутные командные синхронизации
- декомпозиция задач до 8–16 часов
- устранение bottleneck-ролей

**Результат:**
- ⏱ снижение времени встреч на ~75%
- 🚀 ускорение выполнения задач на ~40–50%
- 📊 полная прозрачность статусов и этапов разработки
- 📉 снижение количества блокировок на ~30–40%
- ✅ восстановление управляемости и предсказуемости delivery
