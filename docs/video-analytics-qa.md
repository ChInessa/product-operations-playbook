## 🎥 Внедрение QA-процессов в проекте видеоаналитики

### 📉 Ситуация

Проект по видеоаналитике (Computer Vision) для распознавания событий:

- отсутствие СИЗ (каска, перчатки)
- контроль проведения работ (дренажные работы, длительность)
- контроль параметров (размер факела)

Ключевые проблемы:

- ❌ высокая доля ложных срабатываний модели  
- ❌ команда тестирования без опыта  
- ❌ отсутствие автоматизации тестирования  
- ❌ зависимость от TestRail (недоступен)  
- ❌ отсутствие стабильного регресса  

---

### 🔍 Проблема
 
- тестирование было нерегулярным и несистемным  
- отсутствовала инфраструктура для проверки API  
- не было контроля качества релизов  

---

### ⚙️ Действия

**Организация QA-процессов**
- сформирована и обучена команда тестировщиков  
- выстроен процесс ручного тестирования  

**Миграция инструментов**
- перенос тест-кейсов из TestRail в Zephyr  
- сохранение и структурирование тестовой базы  

**Ручное тестирование**
- регулярное тестирование:
  - dev среда (еженедельно)
  - тестовый стенд  
- покрытие ключевых сценариев  

**Автоматизация**
- разработка API-тестов в Postman  
- запуск автотестов через Newman  
- интеграция в GitLab CI  
- формирование 2 наборов тестов:
  - регрессионные  
  - негативные   

**Улучшение качества**
- выявление проблем валидации  
- постановка задач backend-команде через Jira  

**Документация**
- описание UAT-процесса в Confluence  

---

### 📈 Результат

- ✅ выстроен QA-процесс с нуля  
- 🔁 внедрён регулярный регресс (ежедневный + еженедельный)  
- 🤖 реализована автоматизация API-тестирования   
- 📊 повышена стабильность и качество системы  
- 📚 сформирована база тест-кейсов и документации  

---

### 🧠 Lessons Learned

- тестирование AI ≠ классическое тестирование (важна работа с данными)  
- даже джуниор-команда может быть эффективной при правильной организации  
- автоматизация критична для стабильности и масштабирования
